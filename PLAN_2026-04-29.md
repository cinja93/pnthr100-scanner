## Day 2 — Thursday 2026-04-30 — Phase 4 D1 (Build, all flags off)

### Build sequence

1. **MongoDB collection setup:**
   - Create `pnthr_ibkr_outbox` with indexes on `{ownerId, status, createdAt}`
   - TTL index on `{archivedAt: 1}` with `expireAfterSeconds: 0` for archive collection
   - 90-day cold-archive job (defer cron wiring; run manually until D6+)

2. **`server/ibkrOutbox.js` (new):**
   - `enqueue(db, ownerId, command, request)` — sanity checks + dedup
   - `findPending(db, limit)` — for bridge poller
   - `markExecuting(db, id)` / `markDone(db, id, response)` / `markFailed(db, id, errors)`
   - `findStuck(db)` — EXECUTING > 5 min → STUCK status

3. **`server/ibkrSync.js` updates:**
   - Phase 2 close handler enqueues CANCEL_ORDER for related GTC (gated by `IBKR_AUTO_CANCEL_ON_CLOSE`)
   - Phase 3 open handler enqueues PLACE_STOP at PNTHR Stop (gated by `IBKR_AUTO_PLACE_STOP`)
   - Demo sentinel: `if (ownerId === 'demo_fund') return;` BEFORE any enqueue logic

4. **`server/stopRatchetCron.js` (new):**
   - Daily 4:30 ET cron skeleton
   - Walks active positions, applies tightest-wins, enqueues MODIFY_STOP when PNTHR tighter, silently adopts when IBKR tighter
   - Gated by `IBKR_AUTO_SYNC_STOPS`

5. **`server/exitService.js` updates:**
   - `recordExit` hook enqueues CANCEL_ORDER for related GTC orders post-close (gated by `IBKR_AUTO_CANCEL_ON_CLOSE`)

6. **`server/index.js` admin endpoints:**
   - `GET /api/admin/ibkr-outbox` — queue inspector (recent commands, status, errors)
   - `POST /api/admin/sync-stops` — manual trigger of `stopRatchetCron` logic

7. **`client/src/components/AssistantPage.jsx` updates:**
   - Bridge status badge in top-bar: "Phase 4: Disabled" / "Dry-Run" / "Live (4a+4b)" depending on flags
   - Outbox status section (collapsible, shows recent commands + any STUCK or FAILED)

8. **`pnthr-ibkr-bridge.py` updates:**
   - Outbox poller (every 30s)
   - IB API write methods: `place_order_stop`, `cancel_order`, `modify_stop`
   - Rate limiter: token bucket per symbol (5/min) and global (50/min)
   - All gated by `IBKR_WRITES_ENABLED` env var (default false)
   - `IBKR_WRITES_DRY_RUN` mode: logs commands but does not call IB API

9. **`.env.bridge` flag set:**
   ```
   IBKR_WRITES_ENABLED=false
   IBKR_WRITES_DRY_RUN=true
   IBKR_AUTO_PLACE_STOP=false
   IBKR_AUTO_CANCEL_ON_CLOSE=false
   ```

## Rollback procedures

**Day 1 (UI):** Revert the commit, push. Old Command Center page returns. Zero data loss (frontend-only changes).

**Day 2 (Phase 4 build):** Revert the commit OR set `IBKR_WRITES_ENABLED=false` (instant kill). All flags default off anyway, so revert is rarely needed.

**Day 3-6 (sub-phase enables):** Set the specific `IBKR_AUTO_*` flag back to false in `.env.bridge`. No code change needed. Bridge respects the flag on next sync.

**Outbox catastrophe (e.g., bridge places 50 wrong orders):** `IBKR_WRITES_ENABLED=false` instantly halts ALL bridge writes. Then manually clean up the wrong orders in TWS. Then investigate root cause before re-enabling.
