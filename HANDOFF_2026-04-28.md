## Phase 4 plan ŌĆö what to build tomorrow

**The full architecture lives in `project_phase4_bridge_design.md`.** Summary:

### Architecture
- New MongoDB collection `pnthr_ibkr_outbox` holds queued IBKR write commands.
- Python bridge polls outbox every 30s for `PENDING` commands, executes via IB API, writes status back.
- Node-side hooks enqueue commands at: Phase 2 close (cancel related orders), Phase 3 open (place protective stop), daily 4:30 ET cron (sync ratchets), each IBKR sync (detect partial sells).

### Sub-phases (independently kill-switched)
| Sub | Trigger | IBKR action | Eliminates | Risk |
|---|---|---|---|---|
| 4a | Phase 3 auto-open | PLACE_STOP (STP RTH or STP LMT extended-hours) | NAKED | Low |
| 4b | Phase 2 auto-close | CANCEL all related GTC orders | STALE | Lowest |
| 4c | Daily 4:30 ET cron | MODIFY_STOP with tightest-wins | STOP MISMATCH | Medium |
| 4d | Each sync | Detect partial sells ŌåÆ recordExit | SHARES MISMATCH | Low (read-only inference) |
| 4e | Each sync | Detect adds ŌåÆ record fills | AVG COST DRIFT | Medium (deferred ŌĆö pyramid Lot 2 vs external add disambiguation needed) |

### Universal safety guardrails (every sub-phase honors all of these)
- Master `IBKR_WRITES_ENABLED=false` in `.env.bridge` ships D1.
- Per-action flags: `IBKR_AUTO_PLACE_STOP`, `IBKR_AUTO_CANCEL_ON_CLOSE`, `IBKR_AUTO_SYNC_STOPS`, `IBKR_AUTO_RECORD_PARTIAL_SELL`, `IBKR_AUTO_RECORD_ADD_FILL`. Default all off.

## Critical "do not break" constraints

These are hard rules. Violating any of them risks money or credibility:

1. **Demo account writes are forbidden.** `ownerId === 'demo_fund'` short-circuits before any IBKR write. Hard-coded check, not a flag.
2. **Tightest stop always wins.** PNTHR never loosens a user-set stop. PNTHR adopts TWS values silently when TWS is tighter.
3. **Single canonical close path.** Both manual close (Command Center) and auto-close (Phase 2 IBKR exec) go through `recordExit()` in `exitService.js`. Adding a third write path is a regression.
4. **Refuse to auto-open without a stop.** Phase 3 returns early if `pnthrStop` is null. Never leave a position naked at the system level.
5. **OpEMA naming, never "21 EMA"** for stocks/sectors. Direction index (SPY/QQQ/MDY) is the ONLY 21W permitted; label it "21W Index EMA" so users never confuse the two.
6. **Master kill switch (`IBKR_WRITES_ENABLED=false`) overrides everything.** When false, Phase 4 logic is dead code. New session must verify this is OFF on every deploy.
