## Day 3 enable procedure (Friday 2026-05-01)

**Goal:** turn on 4b. Verify that when a position closes, related GTC stops
auto-cancel within ~30 seconds.

**Steps:**

1. **Add the env vars above** to Render and `.env.bridge` (if you haven't yet).
2. On Render, set `IBKR_AUTO_CANCEL_ON_CLOSE=true`. Save (Render reloads env
   instantly; no redeploy needed).
3. In `.env.bridge`: `IBKR_WRITES_ENABLED=true` and confirm `IBKR_WRITES_DRY_RUN=false`.
4. Restart the bridge via `~/Desktop/Desktop Apps/Start PNTHR Bridge.command`.
5. Watch the bridge log on first iteration:
   - Should print `[OUTBOX] Poller starting — every 30s | writes=ENABLED | dryRun=OFF`
6. **Smoke test (do this before market opens or with a tiny test position):**
   - Manually close any position via Command Center or AssistantRowExpand.
   - Within 30s, the bridge log should show `[OUTBOX] ✓ CANCEL_RELATED_ORDERS … → DONE`.
   - Confirm in TWS: the GTC stop order for that ticker is gone.
7. Throughout the trading day, watch the Assistant bridge badge.
   - **Green badge** = flag enabled, no failures. Good.
   - **Red badge** = something FAILED or STUCK. Click the FAILED banner → check bridge log → fix root cause before continuing.

**Rollback:** flip `IBKR_AUTO_CANCEL_ON_CLOSE=false` on Render. Done. No
restart, no code change — the next close just won't enqueue.

## Hard rules — do not break

(unchanged from yesterday's handoff)

1. **Demo writes hard-disabled.** `ownerId === 'demo_fund'` short-circuits in
   `enqueue()` BEFORE any other check. Verify on every PR that touches
   ibkrOutbox.js or any enqueue call site.
2. **Tightest stop wins.** PNTHR never loosens a user-set stop. Adoption is
   silent (no MODIFY_STOP enqueued).
3. **Single canonical close path.** Both manual close and auto-close go
   through `recordExit()` in `exitService.js`. The 4b hook lives there once.
4. **Refuse to auto-open without a stop.** Phase 3 returns early if
   `pnthrStop` is null.
5. **OpEMA naming, never "21 EMA"** for stocks/sectors. Direction index
   (SPY/QQQ/MDY) is the ONLY 21W permitted; label as "21W Index EMA".
6. **Master kill switch overrides everything.** When
   `IBKR_WRITES_ENABLED=false`, every Phase 4 path is dead code. New session
   must verify this on every deploy.
